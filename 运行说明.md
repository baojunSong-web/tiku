# 错题库系统 - 运行说明

## 一、项目简介

错题库系统是一个帮助管理孩子错题的应用，包含前端和后端两部分：
- **前端**：使用 React + TypeScript + Vite 开发，默认端口 5173
- **后端**：使用 Node.js + Express + SQLite 开发，默认端口 3001

## 二、运行方式

### 方式一：使用 npm 命令启动（推荐给开发者）

#### 步骤 1：安装依赖

首次运行或更新代码后，需要安装依赖：

1. 在项目根目录打开终端
2. 运行以下命令安装所有依赖：
   ```bash
   npm run install
   ```

#### 步骤 2：启动服务

1. 在项目根目录打开终端
2. 运行以下命令启动前后端服务：
   ```bash
   npm run dev
   ```

#### 步骤 3：访问应用

服务启动成功后，在浏览器中访问：
- 错题库系统：http://localhost:5173
- 错题列表页面：http://localhost:5173/questions

### 方式二：使用端口管理脚本启动（推荐给普通用户）

如果您经常遇到端口被占用的问题，可以使用我们提供的端口管理脚本：

#### 步骤 1：启动脚本

1. 直接双击运行 `start_with_port_management.bat` 文件
2. 脚本会自动检查并释放占用的端口，然后启动服务

#### 步骤 2：访问应用

脚本运行成功后，会在终端中显示访问地址，通常是：
- 错题库系统：http://localhost:5173
- 错题列表页面：http://localhost:5173/questions

## 三、常见问题与解决方案

### 1. 端口被占用

**问题**：启动时提示端口已被占用

**解决方案**：
- 使用 `start_with_port_management.bat` 脚本自动释放端口
- 或手动关闭占用端口的程序：
  1. 打开终端
  2. 运行 `netstat -ano | findstr :3001` 查看占用 3001 端口的进程 PID
  3. 运行 `taskkill /pid [PID] /f` 终止该进程
  4. 同样方法处理 5173 端口

### 2. 依赖安装失败

**问题**：运行 `npm run install` 时失败

**解决方案**：
- 确保网络连接正常
- 尝试分别安装前后端依赖：
  ```bash
  # 安装后端依赖
  cd backend && npm install
  
  # 安装前端依赖
  cd frontend && npm install
  ```

### 3. 服务启动后无法访问

**问题**：服务启动成功，但浏览器无法访问

**解决方案**：
- 检查浏览器地址是否正确
- 检查防火墙是否阻止了访问
- 查看终端中的错误日志，根据提示解决问题

## 四、停止服务

### 方式一：使用 npm 命令启动的服务

1. 在启动服务的终端中
2. 按 `Ctrl + C` 组合键
3. 输入 `Y` 并回车确认停止

### 方式二：使用端口管理脚本启动的服务

1. 在启动脚本的终端中
2. 按 `Ctrl + C` 组合键
3. 输入 `Y` 并回车确认停止

## 六、功能说明

- **错题列表**：查看所有添加的错题
- **添加错题**：手动录入或通过图片识别添加错题
- **编辑错题**：修改已添加的错题信息
- **删除错题**：删除不需要的错题
- **筛选功能**：按科目、标签等筛选错题
- **收藏功能**：收藏重要的错题
